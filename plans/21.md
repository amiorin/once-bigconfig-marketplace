# Make home-page count cards clickable

## Reviewed files

- `web/src/pages/index.astro`

## Current state

- The hero already has explicit CTA links to `/packages` and `/applications`.
- The stats section has four cards:
  - `{packageCount}` / `packages`
  - `{applicationCount}` / `applications`
  - `Source` / `on github`
  - `Agent` / `ready`
- All four stats cards are currently plain `<div>` elements, so the package and application count cards are not clickable.

## Goal

Make the entire home-page `# packages` and `# applications` stats cards clickable without changing their visual appearance or the page layout.

## Decisions

- The entire packages card should link to `/packages`.
- The entire applications card should link to `/applications`.
- The `Source / on github` card should remain non-clickable.
- The `Agent / ready` card should remain non-clickable.
- The clickable cards should remain visually unchanged; do not add custom hover styling or other visual treatment.

## Proposed implementation

1. In `web/src/pages/index.astro`, replace the first two stats-card wrapper `<div class="bg-neutral-950 p-6 text-center">` elements with anchors:
   - packages card: `<a href="/packages" ...>`
   - applications card: `<a href="/applications" ...>`
2. Keep the existing typography and spacing so the grid remains visually unchanged.
3. Use the same card classes on the anchors as the current `<div>` cards. Do not add custom hover styles.
4. Add clear accessible labels, for example:
   - `aria-label={`Browse all ${packageCount} packages`}`
   - `aria-label={`Browse all ${applicationCount} applications`}`
5. Leave the `Source / on github` and `Agent / ready` cards as plain, non-clickable `<div>` elements.

## Acceptance criteria

- Clicking anywhere on the packages count card navigates to `/packages`.
- Clicking anywhere on the applications count card navigates to `/applications`.
- Keyboard users can tab to both cards and activate them.
- Existing responsive grid layout and card sizing are preserved.
- Clickable cards look the same as they do now in the default and hover states.
- Astro build still succeeds.
