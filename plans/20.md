## Goal

Update the Marketplace documentation so it matches the new BigConfig model:

- BigConfig packages are no longer Clojure-only.
- Packages can be implemented in TypeScript, Python, or Clojure.
- End users bootstrap and run packages through the `bc-pkg` launcher, available as `npx bc-pkg` on npm and `uvx bc-pkg` on PyPI.
- The launcher pins a GitHub repo/ref to a SHA, copies the package root `run` file, writes a language-native manifest, and forwards future commands to the pinned target.
- `bb.edn` should no longer be presented as the primary user-facing configuration artifact.

### New section: What is BigConfig Launcher

Keep this section high-level and defer internals to the full manual. Cover only the user-facing `bc-pkg` flow:

1. User runs `npx bc-pkg bigconfig-ai/once@typescript ...`, `uvx bc-pkg bigconfig-ai/once@python ...`, or either launcher against `bigconfig-ai/once@clojure ...`.
2. On first run, the launcher pins the GitHub target, copies the package root `run` file, and writes the local manifest needed to run that language.
3. The copied `run` file contains a default profile for the package.
4. Subsequent runs can omit the repo/ref and use the pinned target.

Mention:

- npm package name: `bc-pkg` (`npx bc-pkg ...`).
- PyPI package name: `bc-pkg` (`uvx bc-pkg ...`).
- There is no Clojure implementation of the `bc-pkg` launcher itself, but both launchers can launch Clojure target packages.
